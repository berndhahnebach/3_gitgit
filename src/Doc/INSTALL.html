For Developers

Needed libraries

Because of the heavy use of  third party libraries in FreeCAD its quit hard to set up a development environment so far. You need to download and install a lot of libraries, as there are:

  • Open CasCade from www.opencascade.org in Version 4.0
  • Trolltechs QT GUI library from www.troll.no in Version 2.2.x or higher
  • Python from www.python.org in version 2.1.x or higher

Environment

FreeCAD don't need environment variables beside the ones set by the OCC installer. If they are not set by any reason you can set them by hand:

CSF_MDTVFONTDIRECTORY='E:\Develop\Libs\Cas40\src\FontMFT'
CSF_MDTVTEXTURESDIRECTORY='E:\Develop\Libs\Cas40\src\Textures'
CSF_PLUGINDEFAULTS='E:\Develop\Libs\Cas40\src\StdResource'
CSF_SHMESSAGESTD='E:\Develop\Libs\Cas40\src\SHMessageStd'
CSF_STANDARDDEFAULTS='E:\Develop\Libs\Cas40\src\StdResource'
CSF_UNITSDEFINITION='E:\Develop\Libs\Cas40\src\UnitsAPI\Units.dat'
CSF_UNITSLEXICON='E:\Develop\Libs\Cas40\src\UnitsAPI\Lexi_Expr.dat'
CSF_XSMESSAGESTD='E:\Develop\Libs\Cas40\src\XSMessageStd'

Getting the latest sources

For the source code you can download the src packages from SourceForge or retrieve the latest Version from CVS (which is not guaranteed to compile or run) but your at the bleeding edge of development:

cvs -d:pserver:anonymous@cvs.free-cad.sourceforge.net:/cvsroot/free-cad login
cvs -z3 -d:pserver:anonymous@cvs.free-cad.sourceforge.net:/cvsroot/free-cad checkout src

Building the executables

Windows

On Windows you need Visual C++ 6.0 SP5 to build FreeCAD. There fore you'll find a .DSW file in the src directory. Also provided are Make files for NMAKE.exe if you like.

Linux

There are Make files for Linux, but they are not constantly updated because the main development at the moment happens on Windows2k. We would be happy for active Linux developers and for a configure scritpt.

Building documentation

For building the FreeCAD documentation (Manuals and Framework documentation) run the script MakeDoc.py in the src/Doc directory. It will generate a Doc directory in the top level dir and build an copy the whole documentation.

Contribute

If you want to contribute own code to FreeCAD, following these steps:

  1. Update your source to the newest version by typing:
    cvs -z3 -d:pserver:anonymous@cvs.free-cad.sourceforge.net:/cvsroot/free-cad checkout src
  2. Removing eventually conflicts and compile FreeCAD completely new
  3. Running the build in check to ensure nothing serious is broken by using the test workbench
  4. If the test was successfully generate a patch by typing:
    cvs -z3 -d:pserver:anonymous@cvs.free-cad.sourceforge.net:/cvsroot/free-cad diff src > FreeCAD_yourname_date.patch
  5. Put the patch in the patch tray on SourceForge and maybe drop me a line.

Besides code there are many ways to help us on FreeCAD. You can write bug reports or give us a hint on interesting librarys to use. You can help on the documentation or just tell friends on FreeCAD and and and ......
